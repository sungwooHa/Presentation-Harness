### 라이트 테마 필요성

- 템플릿 기본이 다크 네이비(`#0a1628`)지만, 기존 발표 스타일이나 조직 브랜드가 라이트인 경우 강제 전환 필요
- Phase 1(design-director)에서 청중·맥락에 맞는 테마를 명시적으로 결정해야 함

### 빌드 스크립트 버그 (수정 완료)

1. **셀렉터 불일치**: `section.slide.page` → 실제 HTML은 `div.slide.page` 사용. `.slide.page`로 태그 무관 셀렉터로 수정
2. **`data-act-name` 속성 누락**: `.pact` 클래스 대신 `data-act-name` 속성에서 액트 이름 읽도록 수정
3. **`file://` 한글 경로**: OneDrive 한글 경로 + `file://` 프로토콜 → 브라우저 보안 정책으로 이미지 로드 차단. 임시 HTTP 서버 자동 시작/종료로 해결

### 이미지 배경 사용 원칙

- 이미지를 배경으로 깔 때 반드시 그라데이션 오버레이 필수 (텍스트 가독성)
- 전체 슬라이드의 30% 이하만 이미지 배경 사용 (과용 시 산만)
- `file://` 프로토콜에서는 상대 경로 이미지가 깨질 수 있음 → HTTP 서버 필수 안내

### 한글 경로 `file://` 이슈

- macOS + OneDrive 조합에서 한글 상위 경로를 가진 HTML을 `file://`로 열면 상대 경로 리소스(이미지, 폰트) 로드 실패
- 해결: `python -m http.server` 또는 빌드 스크립트 내장 HTTP 서버 사용
- PPTX 빌드 스크립트에 HTTP 서버 자동 시작 로직 반영 완료
